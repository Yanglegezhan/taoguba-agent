## 核心接口清单

### ✅ 已实现的关键接口

1. **`get_sector_strength_ndays(end_date, num_days=7)`**
   - 用途: 题材筛选
   - 返回: N日板块强度数据

2. **`get_index_intraday(index_code, date=None)`** ⭐ **新增**
   - 用途: 盘面联动分析
   - 返回: 大盘指数分时数据

3. **`get_sector_intraday(sector_code, date=None)`**
   - 用途: 盘面联动分析
   - 返回: 板块分时数据

4. **`get_sector_limit_up_ladder(date=None)`**
   - 用途: 情绪周期分析、容量结构分析
   - 返回: 板块连板梯队数据

5. **`get_realtime_rise_fall_analysis()`**
   - 用途: 情绪周期分析
   - 返回: 全市场涨停数、炸板率、昨日涨停今表现

6. **`get_sector_capital_data(sector_code, date=None)`**
   - 用途: 容量分析
   - 返回: 板块成交额、主力资金等

7. **`get_sector_ranking(date, index=0)`**
   - 用途: 题材筛选、容量分析
   - 返回: 板块涨停股票详情（含成交额、市值）

## 潜在的数据限制和解决方案

### 1. Top5个股数据
**限制**: `get_sector_ranking()`只返回涨停股票，不是板块所有成分股

**解决方案**:
```python
# 从涨停股票中获取Top5（按成交额排序）
def get_top5_stocks(sector_name, date):
    data = crawler.get_sector_ranking(date)
    for sector in data['sectors']:
        if sector['sector_name'] == sector_name:
            stocks = sorted(
                sector['stocks'], 
                key=lambda x: x['成交额'], 
                reverse=True
            )
            return stocks[:5]
```

**评估**: ✅ **可接受** - 涨停股票就是板块最活跃的股票，足够用于容量分析

### 2. 历史炸板率
**限制**: `get_realtime_rise_fall_analysis()`只能获取实时数据

**解决方案**:
- 实时分析时使用该接口
- 历史分析时，LLM可以通过其他指标（连板数、涨停数变化）推断市场情绪

**评估**: ✅ **可接受** - 不影响核心功能
