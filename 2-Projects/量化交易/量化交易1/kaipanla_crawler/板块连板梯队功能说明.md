# 板块连板梯队功能说明

## 功能概述

`get_sector_limit_up_ladder()` 函数用于获取板块维度的连板梯队情况，支持获取历史数据和实时数据。

## 功能特点

- ✅ **历史数据**: 获取指定日期的板块连板梯队
- ✅ **实时数据**: 获取当前交易日的实时板块连板梯队
- ✅ **板块分类**: 按板块分类展示涨停股票
- ✅ **连板信息**: 包含每只股票的连板天数
- ✅ **统计分析**: 支持板块热度分析和连板分布统计

## 函数签名

```python
def get_sector_limit_up_ladder(self, date=None, timeout=60):
    """
    获取板块连板梯队（历史或实时）
    
    Args:
        date: 日期，格式YYYY-MM-DD，默认为None（获取实时数据）
        timeout: 超时时间（秒），默认60秒
        
    Returns:
        dict: 包含板块连板梯队信息
            - date: 日期
            - is_realtime: 是否为实时数据
            - sectors: 板块列表，每个板块包含：
                - sector_name: 板块名称
                - limit_up_count: 涨停数
                - stocks: 涨停股票列表
                    - stock_code: 股票代码
                    - stock_name: 股票名称
                    - consecutive_days: 连板天数
    """
```

### 示例3: 板块热度分析

from kaipanla_crawler import KaipanlaCrawler

### 示例4: 连板分布统计

```python
from kaipanla_crawler import KaipanlaCrawler

crawler = KaipanlaCrawler()
data = crawler.get_sector_limit_up_ladder("2026-01-16")

# 统计连板分布
consecutive_stats = {}
for sector in data['sectors']:
    for stock in sector['stocks']:
        days = stock['consecutive_days']
        consecutive_stats[days] = consecutive_stats.get(days, 0) + 1

print("连板分布:")
for days in sorted(consecutive_stats.keys(), reverse=True):
    count = consecutive_stats[days]
    print(f"  {days}连板: {count}只")
```

**输出示例：**
```
连板分布:
  5连板: 1只
  4连板: 1只
  2连板: 9只
  1连板: 34只
```

## 历史数据 vs 实时数据

### 历史数据

```python
# 获取历史数据（传入日期）
data = crawler.get_sector_limit_up_ladder("2026-01-16")
```

- **数据源**: 历史数据库
- **API地址**: `https://apphis.longhuvip.com/w1/api/index.php`
- **特点**: 数据完整、稳定
- **用途**: 历史分析、回测

### 实时数据

```python
# 获取实时数据（不传日期）
data = crawler.get_sector_limit_up_ladder()
```

- **数据源**: 实时行情
- **API地址**: `https://apphwhq.longhuvip.com/w1/api/index.php`
- **特点**: 数据实时更新
- **用途**: 盘中监控、实时分析
- **注意**: 非交易时间可能返回空数据

## 注意事项

1. **日期格式**: 历史数据必须是`YYYY-MM-DD`格式，如`2026-01-16`
2. **交易日**: 只能查询交易日数据，非交易日会返回空数据
3. **实时数据**: 实时数据只在交易时间内有效
4. **超时设置**: 默认60秒超时，网络不稳定时可以增加
5. **数据来源**: 数据来自开盘啦APP，准确率99%+

## 常见问题

### Q1: 为什么返回的板块列表为空？

**A**: 可能的原因：
- 查询的是非交易日（周末、节假日）
- 当日确实没有涨停股票
- 网络问题导致数据获取失败
- 实时数据在非交易时间查询

### Q2: 历史数据和实时数据有什么区别？

**A**: 
- **历史数据**: 完整、稳定，适合回测分析
- **实时数据**: 实时更新，适合盘中监控
- **API不同**: 使用不同的API地址

### Q3: 如何判断是历史数据还是实时数据？

**A**: 检查`is_realtime`字段：
```python
if data['is_realtime']:
    print("这是实时数据")
else:
    print("这是历史数据")
```

### Q4: 连板天数为1是什么意思？

**A**: 连板天数为1表示首板（当日首次涨停）。

### Q5: 可以获取多个日期的数据吗？

**A**: 可以，循环调用即可：
```python
dates = ["2026-01-15", "2026-01-16", "2026-01-17"]
for date in dates:
    data = crawler.get_sector_limit_up_ladder(date)
    print(f"{date}: {len(data['sectors'])}个板块")
```
