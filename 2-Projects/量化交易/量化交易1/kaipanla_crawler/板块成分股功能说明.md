# 板块成分股功能说明

## 功能概述

新增了获取板块成分股的接口，可以获取指定板块的股票列表及详细信息。

## 接口说明

### 分页机制

**重要发现**：
- 每页返回最多30只股票
- Index参数：0, 30, 60, 90...（每次+30）
- 当返回数量 < 30 时，说明是最后一页
- Count字段是准确的：
  - 第一页：显示核心成分股数量（如40只）
  - 第二页开始：显示所有相关股票总数（如693只）

**测试结果**：
- 化工板块(801235): 693只股票，24页
- 801632板块: 212只股票，8页
- 电力设备(801346): 457只股票，16页

### 1. get_sector_constituent_stocks() - 获取核心成分股

**功能**：获取板块的核心成分股（第一页30只）

**特点**：
- 快速获取，只请求一次
- 返回涨幅最高的30只股票
- Stocks字段包含该板块所有核心成分股代码（30-50只）

**使用场景**：
- 快速查看板块龙头股票
- 获取板块核心成分股代码列表
- 日常监控板块表现

**示例**：
```python
from kaipanla_crawler import KaipanlaCrawler

crawler = KaipanlaCrawler()

# 获取化工板块核心成分股
data = crawler.get_sector_constituent_stocks("801235", "2026-02-10")

print(f"板块代码: {data['plate_id']}")
print(f"返回股票数: {data['total_count']}")  # 30只
print(f"核心成分股代码: {data['stock_codes']}")  # 40只代码

# 遍历股票
for stock in data['stocks']:
    code = stock[0]      # 股票代码
    name = stock[1]      # 股票名称
    concept = stock[4]   # 概念标签
    price = stock[5]     # 最新价
    change = stock[6]    # 涨跌幅
    turnover = stock[7]  # 成交额
    
    print(f"{code} {name}: {price} ({change:+.2f}%)")
```

### 2. get_sector_all_stocks() - 获取所有相关股票

**功能**：分页获取板块的所有相关股票

**特点**：
- 自动分页请求，当返回数量 < 30 时停止
- 包含核心成分股和概念相关股票
- 自动去重
- 可设置最大页数限制
- Count字段准确显示总数

**使用场景**：
- 全面分析板块所有相关股票
- 寻找板块内的潜力股
- 板块轮动分析

**注意事项**：
- 请求时间取决于板块大小（小板块几秒，大板块可能1分钟）
- 建议设置 max_pages 参数避免请求过多
- 返回的股票包含板块所有相关股票

**示例**：
```python
from kaipanla_crawler import KaipanlaCrawler

crawler = KaipanlaCrawler()

# 获取化工板块所有相关股票
data = crawler.get_sector_all_stocks("801235", "2026-02-10")

print(f"核心成分股: {data['core_count']} 只")
print(f"API显示总数: {data['total_count_from_api']} 只")
print(f"实际获取: {data['total_count']} 只")
print(f"获取页数: {data['pages_fetched']} 页")

# 只获取前5页（150只股票）
data = crawler.get_sector_all_stocks("801235", "2026-02-10", max_pages=5)

# 按涨跌幅排序
sorted_stocks = sorted(data['all_stocks'], 
                       key=lambda x: x[6] if len(x) > 6 else 0, 
                       reverse=True)

print("涨幅前10:")
for i, stock in enumerate(sorted_stocks[:10], 1):
    code = stock[0]
    name = stock[1]
    price = stock[5]
    change = stock[6]
    print(f"{i}. {code} {name}: {price} ({change:+.2f}%)")
```

|------|--------|------|------|
| 0 | 股票代码 | 6位代码 | "002455" |

| 4 | 概念标签 | 相关概念 | "光伏、化工" |
| 5 | 最新价 | 当前价格 | 13.48 |
| 6 | 涨跌幅 | 百分比 | 10.04 |
| 7 | 成交额 | 单位：元 | 852462435 |
| 8 | 换手率 | 百分比 | 24.9 |
| 10 | 流通市值 | 单位：元 | 3504149613 |
| 23 | 连板描述 | 如"2连板"、"首板" | "2连板" |
| 24 | 龙头标记 | 如"龙一"、"龙二" | "龙一" |

## 实战案例

### 案例1：找出板块龙头股

```python
from kaipanla_crawler import KaipanlaCrawler

crawler = KaipanlaCrawler()

# 获取化工板块核心成分股
data = crawler.get_sector_constituent_stocks("801235", "2026-02-10")

# 筛选涨停股票
limit_up_stocks = []
for stock in data['stocks']:
    change_pct = stock[6]
    if change_pct >= 9.9:  # 涨停
        limit_up_stocks.append({
            'code': stock[0],
            'name': stock[1],
            'price': stock[5],
            'change': stock[6],
            'board': stock[23],  # 连板描述
            'leader': stock[24]  # 龙头标记
        })

print(f"涨停股票: {len(limit_up_stocks)} 只")
for s in limit_up_stocks:
    print(f"{s['code']} {s['name']}: {s['change']:.2f}% {s['board']} {s['leader']}")
```

## 性能建议

1. **优先使用 get_sector_constituent_stocks()**
   - 如果只需要核心成分股，使用此接口
   - 速度快，只请求一次

2. **谨慎使用 get_sector_all_stocks()**
   - 会发起多次请求（可能50+次）
   - 建议设置 max_pages 参数限制页数
   - 添加缓存机制避免重复请求

3. **批量获取时添加延迟**
   ```python
   import time
   
   plates = ["801235", "801346", "801225"]
   for plate_id in plates:
       data = crawler.get_sector_constituent_stocks(plate_id)
       # 处理数据...
       time.sleep(1)  # 延迟1秒
   ```

## 常见问题

### Q1: Count 字段是什么意思？

A: Count 字段是准确的总数：
- 第一页(Index=0)：显示核心成分股数量（如40只）
- 第二页开始(Index=30+)：显示所有相关股票总数（如693只）

### Q2: 如何判断是否获取了所有股票？

A: 当返回的股票数量 < 30 时，说明是最后一页。例如：
- 化工板块：第24页返回3只，停止
- 801632板块：第8页返回2只，停止
- 电力设备：第16页返回7只，停止

### Q3: Stocks 字段和 list 字段有什么区别？

A:
- `Stocks` 字段：只在第一页返回，包含所有核心成分股代码（30-50只）
- `list` 字段：每页返回最多30只股票的详细信息

### Q4: 为什么第一页的 Stocks 字段有40个代码，但 list 只返回30只？

A: 
- Stocks 字段包含该板块的所有核心成分股代码（40只）
- list 字段按涨幅排序，每页最多返回30只
- 剩余的10只核心成分股会在第二页返回

### Q5: 如何获取板块代码？

A: 可以通过以下方式获取：
```python
# 方法1: 通过板块排行接口
data = crawler.get_sector_ranking("2026-02-10")
for sector in data['sectors']:
    print(f"{sector['sector_code']} {sector['sector_name']}")

# 方法2: 通过板块连板梯队接口
data = crawler.get_sector_limit_up_ladder("2026-02-10")
for sector in data['sectors']:
    print(f"{sector['sector_code']} {sector['sector_name']}")
```
