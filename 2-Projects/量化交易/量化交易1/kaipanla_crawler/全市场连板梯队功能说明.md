# 全市场连板梯队功能说明

## 功能概述

`get_market_limit_up_ladder()` 函数用于获取全市场的连板梯队情况，支持实时和历史数据查询，提供完整的市场连板分布数据。

## 功能特点

- ✅ **实时和历史**: 支持获取实时数据和任意历史交易日数据
- ✅ **全市场覆盖**: 获取所有涨停股票的连板情况
- ✅ **连板梯队**: 按连板天数分类展示（首板、2连板、3连板等）
- ✅ **反包板识别**: 单独记录反包板股票（不计入连板梯队）
- ✅ **打开高度标注**: 单独记录打开高度标注股票
- ✅ **统计分析**: 提供总涨停数、最高连板、连板分布等统计信息

## 函数签名

```python
def get_market_limit_up_ladder(self, date=None, timeout=60):
    """
    获取全市场连板梯队（历史或实时）
    
    Args:
        date: 日期，格式YYYY-MM-DD，默认为None（获取实时数据）
        timeout: 超时时间（秒），默认60秒
        
    Returns:
        dict: 包含全市场连板梯队信息
            - date: 日期
            - is_realtime: 是否为实时数据
            - ladder: 连板梯队数据
                - 1: 首板股票列表
                - 2: 2连板股票列表
                - 3: 3连板股票列表
                - ...
            - broken_stocks: 反包板股票列表
            - height_marks: 打开高度标注股票列表
            - statistics: 统计信息
                - total_limit_up: 总涨停数
                - max_consecutive: 最高连板
                - ladder_distribution: 连板分布
    """
```

## 核心概念

### 什么是反包板？

反包板是指股票在涨停后打开，然后再次封板的情况。这类股票：
- 在API中用`Tip=0`标识
- 从`tips`字段解析连板信息（如"3天2板"表示3天内2次涨停）
- **不计入连板梯队**（因为不是连续涨停）
- 单独记录在`broken_stocks`列表中

### 什么是打开高度标注？

打开高度标注是指曾经达到过某个连板高度但后来打开的股票：
- 在API中用`Tip=9`标识
- **不计入连板梯队**
- 单独记录在`height_marks`列表中

### 实时数据 vs 历史数据

- **实时数据**: 不传`date`参数，获取当前交易日的实时连板情况
  - 数据来源: `https://apphwhq.longhuvip.com/w1/api/index.php`
  - `is_realtime=True`
  - 适用于盘中监控

- **历史数据**: 传入`date`参数，获取指定交易日的连板情况
  - 数据来源: `https://apphis.longhuvip.com/w1/api/index.php`
  - `is_realtime=False`
  - 适用于复盘分析

### 连板梯队分类

- **首板（1连板）**: 当日首次涨停
- **2连板**: 连续2天涨停
- **3连板**: 连续3天涨停
- **N连板**: 连续N天涨停

### 连板率计算

```python
连板率 = (连板股票数 / 总涨停数) × 100%
连板股票数 = 总涨停数 - 首板数
```

连板率反映市场的持续性和强度：
- 连板率高：市场情绪强，资金持续性好
- 连板率低：市场情绪弱，资金持续性差

## 注意事项

1. **日期格式**: 必须是`YYYY-MM-DD`格式，如`2026-01-16`
2. **实时数据**: 不传`date`参数获取实时数据，传`date`参数获取历史数据
3. **交易日**: 历史数据只能查询交易日，非交易日会返回错误
4. **数据完整性**: 返回的是全市场数据，包含所有涨停股票
5. **反包板**: 反包板股票不计入连板梯队统计
6. **打开高度标注**: 打开高度标注股票不计入连板梯队统计
7. **超时设置**: 默认60秒超时，网络不稳定时可以增加
8. **数据来源**: 数据来自开盘啦APP，准确率99%+

## 常见问题

### Q1: 为什么返回的涨停数为0？

**A**: 可能的原因：
- 查询的是非交易日（周末、节假日）
- 当日确实没有涨停股票
- 网络问题导致数据获取失败

### Q2: 反包板和连板有什么区别？

**A**: 
- **连板**: 连续多日涨停，中间没有打开
- **反包板**: 涨停后打开，然后再次封板
- 反包板不计入连板梯队统计

### Q3: 如何判断市场强弱？

**A**: 可以通过以下指标：
- 总涨停数：越多越强
- 最高连板：越高越强
- 连板率：越高越强
- 反包板数：越少越强

### Q4: 数据更新频率是多少？

**A**: 历史数据在交易日收盘后更新，通常在当日晚上可以查询到当日数据。

### Q5: 可以获取实时数据吗？

**A**: 可以！不传`date`参数即可获取实时数据：
```python
# 获取实时数据
data = crawler.get_market_limit_up_ladder()

# 获取历史数据
data = crawler.get_market_limit_up_ladder("2026-01-16")
```
