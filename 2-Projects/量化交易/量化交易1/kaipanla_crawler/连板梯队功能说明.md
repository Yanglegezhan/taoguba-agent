# 连板梯队功能说明

## 功能概述

`get_consecutive_limit_up()` 函数用于获取指定日期的连板梯队情况，包括：
- 最高连板高度
- 最高连板个股名称（多个用`/`分隔）
- 最高连板个股题材（多个用`/`分隔，同一个股的多个题材用`、`分隔）
- 完整的连板梯队数据（2连板、3连板、4连板等）

## 函数签名

```python
def get_consecutive_limit_up(self, date=None, timeout=60):
    """
    获取指定日期的连板梯队情况
    
    Args:
        date: 日期，格式YYYY-MM-DD，默认为当前日期
        timeout: 超时时间（秒），默认60秒
        
    Returns:
        dict: 包含连板梯队信息
            - date: 日期
            - max_consecutive: 最高连板高度
            - max_consecutive_stocks: 最高连板个股名称（多个用/分隔）
            - max_consecutive_concepts: 最高连板个股题材（多个用/分隔）
            - ladder: 连板梯队详细数据
                - 2: 二连板股票列表
                - 3: 三连板股票列表
                - 4: 四连板股票列表
                - ...
    """
```

## 使用示例

```python

### 示例4: 统计连板分布

# 统计各连板高度的股票数量
print("连板分布:")
for consecutive in sorted(data['ladder'].keys(), reverse=True):
    count = len(data['ladder'][consecutive])
    print(f"  {consecutive}连板: {count}只")

# 计算总连板数
total = sum(len(stocks) for stocks in data['ladder'].values())
print(f"\n总连板数: {total}只")
```

**输出示例：**
```

  4连板: 2只
  3连板: 5只
  2连板: 15只

总连板数: 22只
```

## 字段说明

### 顶层字段

| 字段 | 类型 | 说明 |
|------|------|------|
| date | str | 日期，格式YYYY-MM-DD |
| max_consecutive | int | 最高连板高度 |
| max_consecutive_stocks | str | 最高板个股名称，多个用`/`分隔 |
| max_consecutive_concepts | str | 最高板个股题材，多个用`/`分隔 |
| ladder | dict | 连板梯队详细数据 |

### ladder字段（股票信息）

| 字段 | 类型 | 说明 |
|------|------|------|
| 股票代码 | str | 6位股票代码 |
| 股票名称 | str | 股票名称 |
| 连板天数 | int | 连续涨停天数 |
| 题材 | str | 主要题材 |
| 概念 | str | 相关概念，多个用`、`分隔 |

## 注意事项

1. **日期格式**: 必须是`YYYY-MM-DD`格式，如`2026-01-19`
2. **交易日**: 只能查询交易日数据，非交易日会返回空数据
3. **超时设置**: 默认60秒超时，网络不稳定时可以增加
4. **题材分隔**: 
   - 不同个股之间用`/`分隔
   - 同一个股的多个题材用`、`分隔
5. **数据来源**: 数据来自开盘啦APP，准确率99%+

## 错误处理

```python
from kaipanla_crawler import KaipanlaCrawler

crawler = KaipanlaCrawler()

try:
    data = crawler.get_consecutive_limit_up("2026-01-19")
    
    if data['max_consecutive'] == 0:
        print("当日无连板数据（可能是非交易日）")
    else:
        print(f"最高板: {data['max_consecutive']}连板")
        
except Exception as e:
    print(f"获取数据失败: {e}")
```

## 性能优化

函数会从高到低尝试获取连板数据（最多尝试到20连板），找到最高板后会继续获取所有连板数据。如果只需要最高板信息，可以在获取后立即返回：

```python
# 只获取最高板信息（不获取完整梯队）
data = crawler.get_consecutive_limit_up("2026-01-19")
max_consecutive = data['max_consecutive']
max_stocks = data['max_consecutive_stocks']
max_concepts = data['max_consecutive_concepts']
```

## 常见问题

### Q1: 为什么返回的最高板是0？

**A**: 可能的原因：
- 查询的是非交易日（周末、节假日）
- 当日确实没有连板股票
- 网络问题导致数据获取失败

### Q2: 题材字段为空怎么办？

**A**: 部分股票可能没有明确的题材标签，这是正常现象。可以通过概念字段获取相关信息。

### Q3: 如何判断数据是否有效？

**A**: 检查`max_consecutive`字段：
```python
if data['max_consecutive'] > 0:
    # 数据有效
    pass
else:
    # 无数据或数据无效
    pass
```

### Q4: 可以获取历史数据吗？

**A**: 可以，只要指定历史日期即可：
```python
data = crawler.get_consecutive_limit_up("2025-12-31")
```
