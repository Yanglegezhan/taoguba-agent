### 核心计算指标

```python
# 晋级率计算
晋升率 = 昨日3板以上股今日晋级成功数 / 昨日3板以上股总数

# 情绪周期判断标准
冰点期: 晋级率 < 10% 且 出现核按钮
退潮期: 晋级率 < 20% 且 高位股集体补跌
主升期: 晋级率 > 50% 且 涨停数量多
修复期: 晋级率回升 + 资金回流

# 板块主线识别
主线标准: 涨停数 ≥ 5 且 涨幅 > 3% 且 有梯队支撑
支线标准: 涨停数 < 5 或 事件驱动
```

## 🎯 关键技术难点

### 1. 晋级率计算准确性
- 问题：需要准确获取昨日3板以上个股
- 解决：使用开盘啦的连板梯队接口，结合历史数据

### 2. 板块主线识别
- 问题：如何区分主线和支线
- 解决：设置量化标准（涨停数≥5，涨幅>3%）+ 质化判断（逻辑硬核度）

### 3. 个股逻辑提取
- 问题：从概念标签提取有意义的逻辑
- 解决：建立关键词库（龙头、量产、突破、政策、订单等）

### 4. 情绪周期定位
- 问题：多维度判断，容易混淆
- 解决：建立判断树，按优先级依次判断
